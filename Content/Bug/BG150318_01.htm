Web APIでプロジェクトマネージャを設定すると、サーバタスクの実行に失敗する場合がある

更新日:

2015/3/24

対象バージョン:

3.6以降

対象エディション:

Professional / Standard

現象

Web APIを用いてプロジェクトを新規作成したりプロジェクトマネージャを設定すると、サーバタスクの［プロジェクト情報の更新］でエラーが発生する場合があります。
これにより、分析機能で使用する［プロジェクト情報の更新］のデータが更新されません。
※［プロジェクト情報の更新］のデータを使用する分析機能についてはこちらをご参照ください。

発生条件

本現象は以下の条件で発生します。

条件1

  • Web API のPOST /projects（プロジェクトの新規作成）を用いてプロジェクトを新規作成する
  • プロジェクトマネージャに、Web API を実行するアカウントとは別のアカウントを設定する

条件2

  • Web API のPUT /projects/projectId（プロジェクト情報の更新）を用いてプロジェクトマネージャを更新する
  • プロジェクトマネージャに、プロジェクトメンバに登録されていないアカウントを設定する

回避方法

以下の方法のいずれかを行うことにより、本現象を回避できます。
また、本現象が発生した場合でも同様の方法を行うことにより解消できます。
これにより、サーバタスクの［プロジェクト情報の更新］が成功し、分析機能で扱う情報が最新のものになります。

  • Web APIにてプロジェクトマネージャに設定するアカウントをプロジェクトメンバに追加する
  • Web APIを使用せず、TimeTracker FXのアプリケーションからプロジェクトマネージャを設定する
