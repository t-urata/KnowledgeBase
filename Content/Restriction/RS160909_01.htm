更新日:

2016/9/9

対象バージョン:

3.7以降

対象エディション:

Professional / Standard

現象

Web Editionでタイムシートを開いた時に一部の実績データが表示されない。

発生条件

一週間の実績データが100件を超えた場合に発生します。
タイムシートの表示単位を「日」に変更しても、当該週の実績データ件数が100を超えると発生します。

回避方法

Web Editionのサーバ上で定義されているデータ取得の上限数を、一週間の実績データ数を考慮した値に変更します。上限数を無制限にする場合は0を設定します。
変更方法は以下のとおりです。

【変更方法】

Web APIのインストールフォルダ直下にあるファイル「Web.config」をテキストエディタで開き、以下の赤字の箇所を修正して保存します。

----------------------------------------------------------------------

<!-- APIによるレコード取得件数の設定値 -->
<add key="DefaultLimit" value="100" />

----------------------------------------------------------------------

Web APIのインストールフォルダ名はIISマネージャーで確認することができます。

上記画面の場合、Web APIのインストールフォルダパスは以下になります。

C:\inetpub\wwwroot\TimeTrackerFXWebAPI3.9.0
