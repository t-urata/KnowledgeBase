SQL Server 2008のsaアカウントのパスワードを忘れた時など、パスワードを変更する方法をご紹介します。
パスワードの変更は、「Microsoft SQL Server Management Studio」を使って強制的にパスワードを変更します。
※この変更方法は、SQL Server 2005も同様な操作で変更可能です。

設定方法

  1. 管理者権限を持つユーザでサーバマシンにログインします。
    本操作は、SQL Serverの「Windows認証」を利用してsaアカウントのパスワードを変更します。
    サーバマシンのAdministratorまたはAdministratorsグループに所属するユーザにてログインします。 
  1. Windows［スタート］メニュー－［すべてのプログラム］－［Microsoft SQL Server 2008］から「Microsoft SQL Server Management Studio」を起動します。
    TimeTracker FXに同梱される「SQL Server Express」には、「Microsoft SQL Server Management Studio」が含まれません。
    Microsoft社のホームページよりダウンロードいただき、別途インストールを行ってください。
  2. 認証モードに「Windows認証」を選択し、SQL Serverへ接続します。

  3. ［オブジェクトエクスプローラ］の［セキュリティ］－［ログイン］より「sa」アカウントを選択し、［プロパティ］を開きます。



  4. ［全般］タブの［パスワード］に新しいパスワードを入力します。



  5. ［状態］タブの［設定］情報を確認します。
    • データベースエンジンへの接続する権限　・・・　許可
    • ログイン　・・・　有効
  6. オブジェクトエクスプローラのデータベース名からSQL Serverのサービスを再起動します。





  7. 「Microsoft SQL Server Management Studio」を再度起動します。

  8. 認証モードに「SQL Server認証」を選択し、以下の情報にてSQL Serverへ接続し、パスワードが変更されたことを確認します。
    • ログイン　　　・・・　sa
    • パスワード　 ・・・　手順５にて設定した新しいパスワード


 

これで、パスワードの変更が完了です。

SQL Serverの設定確認

手順9の後、TimeTracker FXのデータベースを作成、登録を行う前に下記のSQL Serverの設定が正しく設定されていることをご確認下さい。

  1. SQL Serverのサーバ認証が「 SQL Server認証とWindows認証」モードとなっていること  

    ［データベースのプロパティ］－［セキュリティ］から［サーバ認証］が「混合モード（SQL Server認証モードとWindows認証モード）」であることを確認する。

    ※変更した設定情報を適応するには、SQL Serverのサービスの再起動が必要です。

  2. SQL Serverへのリモート接続が許可されていること  

    ［データベースのプロパティ］－［接続］から［リモートサーバ接続］の設定が許可されていることを確認する。

    ※変更した設定情報を適応するには、SQL Serverのサービスを再起動の必要です。

  3. 通信プロトコルのTCP/IPが有効になっていること

    ［Microsoft SQL Server 200*］－［構成ツール］－［SQL Server 構成マネージャー］から
    ［SQL Server ネットワークの構成］－［TIMETRACKERFXのプロトコル］設定の［TCP/IP］が「有効」でことを確認する。
