SQL Server 2005からバージョンアップする際の注意点

更新日:

2016/3/11

対象バージョン:

3.5以前

対象エディション:

Professional / Standard

 

概要

2016年4月12日に、Microsoft SQL Server 2005（以下「SQL Server 2005」）のサポートが終了します。
SQL Server 2005を利用中の場合は サポート終了を機に、SQL Serverをバージョンアップすることをお奨めします。

TimeTracker FXのデータベースの移行に必要な設定や手順については、「TimeTracker FX データベース移行・復旧ガイド」でご紹介していますが、
以下の２つの条件を満たす場合、ガイドでご紹介している手順で移行することができません。

  • SQL Server 2005上でTimeTracker FX 3.5以前のバージョンを利用している
  • SQL Server 2012以降へバージョンアップする

ここでは、TimeTracker FX 3.5以前のバージョンのデータベースを、SQL Server 2012以降で利用するために必要な設定や手順をご紹介します。

理由と移行手順

TimeTracker FX 3.5以前では、SQL Serverの互換性レベル 80のデータベースを作成します。
SQL Server 2012以降のデータベースは互換性レベル 80をサポートしていないため、TimeTracker FX をご利用になれません。

データベースの互換性レベルを、SQL Server 2012以降のデータベースが対応しているレベルに上げる必要があります。
TimeTracker FX 3.8にアップグレードすることで、SQL Server　2012以降で利用可能な互換性レベルとなります。

手順の詳細は[SQLServer2005MigrationGuide.pdf]をご覧ください。
