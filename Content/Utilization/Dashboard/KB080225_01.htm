推移モニタの具体的な使用方法をご紹介します。

推移モニタは、ダッシュボード･アナリスト･プランナーから確認できます。

  • FX 3.1以降のバージョンでは、ダッシュボードの[推移モニタ]は[プロジェクト推移モニタ]という名称になります。

 

はじめに

推移モニタは、プロジェクトやプロジェクト内の特定のタスク･タスクパッケージの、進捗･工数･コストなどの情報を週ごとに集計し、折れ線グラフで表示できます。

推移モニタはダッシュボード･アナリスト･プランナーで使用できます。それぞれ以下の特徴があります。

  • ダッシュボード
    プロジェクトや注目する作業の推移を常に表示しておき、作業が順調であることを確認できます。

  • アナリスト
    複数のプロジェクトを確認中に、気になったプロジェクトの作業状況の変化を確認できます。

  • プランナー
    問題が発生したプロジェクトでどの部分がいつ頃から問題を生じていたか確認できます。

 

例として、［TimeTrackerの開発］プロジェクトとその中の［作成］タスクパッケージの、計画工数と実績工数をモニタする方法をご紹介します。

TimeTracker FXを4週間運用し、週単位の動向を確認します。

  1. サーバタスクの設定を確認する

  2. 推移モニタで確認する対象を公開ノードに設定する

  3. TimeTracker FXを運用する

  4. プランナーでプロジェクト全体の推移を確認する

  5. ダッシュボードで［作成］タスクパッケージの推移を確認する

 

サーバタスクの設定を確認する

推移データを作成するサーバタスクの設定を確認します。

本手順では、TimeTracker FX 2.5以降を例にして説明します。

TimeTracker FX 2.0.xをご利用の場合は、 こちらをご覧ください。

  1. Windows［スタート］-［すべてのプログラム］-［TimeTracker FX］-［Server Manager］を選択します。

  2. Server Managerの［データベースの管理］の［サーバタスク処理状態］欄で運用中のデータベースが［稼働中］となっていることを確認します。

  • ［サーバタスク処理状態］が［停止中］となっている場合はこちらをご覧ください

  1. Server Managerの［データベースの管理］で運用中のデータベースの［データベース］欄をクリックします。

  2. プロパティ画面で、［データベースの管理］の［サーバタスクを設定する］をクリックします。

  3. ［サーバタスク一覧］の［推移データの作成］の［次回実行日時］が翌日以降に設定されていることを確認します。

  • 設定した［次回実行日時］にサーバマシンが稼働していない場合は、サーバタスクが実行されません。

  1. 集計する曜日と時間を変更する場合は、［曜日］･［時刻］欄のリストから任意の値を設定します。

  2. ［データベースに反映］をクリックすると、変更後の［曜日］･［時刻］が登録されます。

  • ［次回実行日時］は、変更前の［次回実行日時］以降の日時に再設定されます。

 

推移モニタで確認する対象を設定する

推移モニタで確認する対象を公開ノードに設定します。

  • 公開ノードに設定する以前のデータは推移モニタに表示できません。

 

  1. Windows［スタート］-［すべてのプログラム］-［TimeTracker FX］-［TimeTracker FX］を選択します。

  2. TimeTracker FXのログイン画面を表示しますので、ログインします。

  4. ［プロジェクトを開く］ダイアログから、［TimeTrackerの開発］プロジェクトを選択します。

  5. ［作成］タスクパッケージを選択します。

  6. ［編集］メニューの［公開ノードに設定］を選択します。

  • FX 3以降は、［スケジュール］メニューの［公開ノードに設定］を選択します。

 

TimeTracker FXを運用する

TimeTracker FXを運用し、実績を入力します。

推移モニタのデータは毎週集計されます。

  • 推移モニタのデータは、集計時点のデータです。後日追加した実績は、過去の推移データに反映されません。

 

  1. ［ファイル］メニューの［サーバタスク］を選択します。

  2. ［サーバタスク］ダイアログを表示しますので、［フィルタ条件］の［+］をクリックしてください。

  • TimeTracker FX 2.0.xの場合は、手順3に進んでください。

  1. ［処理済みの要求も表示］にチェックをいれます。

  2. ［推移データの作成］が計画通り処理されていることを確認します。

 

プランナーで推移モニタを確認する

プランナーの［推移モニタ］ドッキングウィンドウで、推移モニタを表示します。
