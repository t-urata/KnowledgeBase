Web APIの概要

TimeTracker FX Web API(以下Web API)は、Internet Information Servicese(以下IIS)上で動作するAPIです。
Web APIを使用するとWebサーバを介してTimeTracker FXのデータを操作することができます。
そのため、ユーザはTimeTracker FXがインストールされていないクライアントPCからでもTimeTracker FXのデータを閲覧したり、更新したりできます。

Web APIを使用するための構成

Web APIを使用するためには、以下の構成を用意する必要があります。

  • TimeTracker FX サーバ ： TimeTracker FXのサーバマシン
  • Web サーバ ： IISとWeb APIをインストールしたサーバマシン
  • クライアントPC

各要素の関係は、下図のようになります。

Web APIのサンプルツールについて

ナレッジベースの本カテゴリ内ではWeb APIのサンプルツールを紹介しています。
Web APIを利用するためのご参考にしてください。
これらのサンプルツールは、以下の方法で利用できます。

Webサーバ上に格納する

Webサーバに格納し、クライアントPCからWebサーバ上のサンプルツールを利用します。
全ての利用者が一つのツールを使用するため、ツールの管理が容易です。

  • サンプルツールをWebサーバ上で実行するためには、クライアントPCのセキュリティ設定を変更する必要があります。

    • 以下の2点のセキュリティ設定を行います。
    ここではInternet Explorer 8での設定例を紹介します。
    その他のブラウザでの設定については、利用するブラウザのヘルプをご確認ください。

    1. サンプルツールを公開するWebサーバを「信頼済みサイト」に登録する。
      1. ［ツール］メニューの［インターネット オプション］を選択する。
      2. ［インターネット オプション］ダイアログの［セキュリティ］タブにて、［信頼済みサイト］を選択する。
      3. ［サイト］ボタンをクリックして表示する［信頼済みサイト］ダイアログにて、「このWebサイトをゾーンに追加する」の箇所にサンプルツールを公開するWeb サイトのURLを入力し、［追加］をクリックする。

    2. 「信頼済みサイト」のセキュリティゾーンを変更し、ファイル出力時に確認のダイアログを表示する。
      1. セキュリティのレベルを［中高］以下に設定し、［保護モードを有効にする］のチェックを外す。
      2. ［レベルのカスタマイズ］をクリックして［セキュリティ設定-信頼されたゾーン］ダイアログを開く。
      3. ［ActiveX コントロールとプラグイン］－［スクリプトを実行しても安全だとマークされていない ActiveX コントロールの初期化とスクリプトの実行］項目の設定を「ダイアログを表示する」に変更する。

    クライアントPCのセキュリティ設定を変更することが困難な場合は、後述する［クライアントPC上に格納する］の利用方法をご検討ください。


  • Web サーバAとWebサーバBは同一のWebサーバにして運用することもできます。

クライアントPC上に格納する

クライアントPCに格納し、クライアントPC上のサンプルツールを利用します。
利用者ごとにツールをカスタマイズして利用する場合や、［Webサーバ上に格納する］で紹介したクライアントPCのセキュリティ設定が実施困難な場合にご検討ください。
