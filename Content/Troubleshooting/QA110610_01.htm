このトピックの対象

TimeTracker FX 評価版

TimeTracker FX 3以降の製品版

 

症状

［autorun.exe］または、TimeTracker FXに同梱される［SQLEXPR_JPN.EXE］よりMicrosoft SQL Server Expressをインストールすると、

以下のエラーが発生し、SQL Server Expressのインストールに失敗する場合があります。

 

  • ［autorun.exe］の場合

    • エラーの詳細は こちらをご参照ください。

 

  • ［SQLEXPR_JPN.EXE］の場合

 

 

原因と対策

本現象は、SQL Server Expressのインストールを実施したマシンに、［MSXML6 SP2］がインストールされていることで、

SQL Server Expressのインストールパッケージに含まれる［MSXML6］のインストールに失敗し、

［SQL Server データベースサービス］のインストールが中断されてしまいます。

 

本現象は、Microsoft社の修正モジュール（MSXMLFix.EXE）にて修正を実施することで、回避することができます。

以下の手順にてご対応ください。

 

【手順】

    1. 以下のMicrosoft社のサイトから修正モジュール（MSXMLFix.EXE）をダウンロードします。

http://support.microsoft.com/kb/968749/ja

    1. ダウンロードしたモジュールを展開し、MSXMLFix.EXE を実行すると、使用許諾を求められるので [Yes] をクリックします。
      Winzip が起動されますので、任意のフォルダへ展開します。(例: c:\temp)
    2. MSXML6.MSP と MSXML_FIX.VBS が同じフォルダへ展開されたことを確認します。
    3. コマンド プロンプトを起動します。
    4. ファイルを展開したフォルダへ移動します。(例: cd c:\temp)
    5. 以下のコマンドを入力し、スクリプトを実行します。
      コマンド：cscript MSXML_FIX.VBS
    6. SQL Server Express のインストールを実行します。

 

 

上記の対処方法にて現象が解消しない場合は、お問い合わせに必要な情報を添えて、こちらまでご連絡ください。
