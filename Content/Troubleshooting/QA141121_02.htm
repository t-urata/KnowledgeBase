Web Editionのログインに失敗する

更新日:

2014/12/2

対象バージョン:

TimeTracker FX 3.7以降

対象エディション:

Professional / Standard

 

TimeTracker FX Web Editionのログイン時にエラーが発生し、ログインに失敗する場合があります。
この時の対処方法を紹介します。

症状

TimeTracker FX Web Editionのログイン画面で、以下のようなエラー画面が表示されてログインができません。
エラーの内容によって、画面の表示は異なります。

本現象は以下の場面で発生します。(必ず発生するわけではありません)

・ Web サーバにTimeTracker FX Web Editionをインストールして、クライアントPCからWeb Editionにアクセスする場合

原因と対策

本現象が発生する主な原因を列挙します。
以下の内容を順番にご確認ください。

  1. ログイン情報の入力ミス
  2. 必須動作環境の不足
  3. Web Editionのセットアップミス
  • なお、本コンテンツで利用するIISは、IIS7を利用しております。
    このため、ご利用いただくIISのバージョンによっては、操作画面や方法が若干異なる場合があります。

これらの原因が該当しない場合は、ご利用中のサーバ・クライアントの環境(OSやIISの種類、ネットワーク環境など)を含む、
お問い合わせに必要な情報をこちらまでご連絡ください。

 

1.ログイン情報の入力ミス

TimeTracker FX Web Editionをご利用いただくためは、データベースに登録されている有効なアカウントにてログインしていただく必要があります。
このログイン画面でご入力いただくログイン情報に誤りがある場合、TimeTracker FXにログインすることができず、エラーが発生します。

利用いただいたログイン情報について、以下の内容に誤りがないかをご確認ください。

1.1 ログインアカウントの確認

【確認内容】

Web Editionは、TimeTracker FXデスクトップ版を利用可能なアカウントのみが利用できます。

ログインに利用したアカウントは、TimeTracker FX デスクトップ版を利用可能なアカウントかどうかを確認します。

【確認方法】

  1. TimeTracker FX デスクトップ版が利用可能な環境で、対象アカウントにてデスクトップ版にログインする。
  2. ［実績を入力する］からタイムシートを起動し、実績の入力・閲覧が可能であることを確認する。

【確認結果と対処方法】

  • デスクトップ版にログインできない場合

対象のアカウントがAdminisratorツール－［アカウントの編集］に登録されていない、または「有効」に設定されていない可能性があります。

Administratorツール－［アカウントの編集］にて、該当アカウントの登録と［無効］フィールドの設定状態をご確認いただくか、
他のログイン可能なアカウントでのログインをお試しください。

  • デスクトップ版にログインできる場合

対象アカウントは、TimeTracker FX Web Editionを利用可能なアカウントです。
今回確認した「アカウント」で、再度Web Editionへのログインをお試しください。

現象が解消されない場合には、「1.2 ログイン名、パスワードの確認」を確認してください。

 

1.2 ログイン名、パスワードの確認

【確認内容】

Web Editionへのログインでは、TimeTracker FXデスクトップ版と同様にログイン名の大文字・小文字を判別します。

このため、入力されたログイン名、パスワードに誤りがないかを確認します。

【確認方法】

  1. キーボードのCapsLockなどが有効になっていないを確認する。
  2. ログイン名、パスワードに誤入力がないかを確認する。
  • デスクトップ版でActive Directory連携機能が有効な場合、入力するパスワードは連携するActive Directoryで有効なパスワードとなります。

【確認結果と対処方法】

  • ログイン名、パスワードに誤りがあった場合

正しいログイン名、パスワードを入力し、再度ログインをお試しください。

  • ログイン名、パスワードに誤りがない場合

Web Editionを利用するための必須動作環境の不足やセットアップミスの可能性があります。
次項以降の内容をご確認ください。

 

 

2.必須動作環境の不足

TimeTracker FX Web Editionでは、「TimeTracker FX Web API」の機能を使用しており、
Web Editionを利用するためには、「TimeTracker FX Web API」がインストールされたWebサーバが必要となります。

このため、Web Editionを提供するためにご用意いただくWebサーバに、Web APIがインストールされていない場合には、エラーが発生します。
また、WebサーバとTimeTracker FXのサーバマシンがネットワーク接続できない環境では、データベース上から情報を取得することができず、エラーとなります。

ご用意いただきました、WebサーバおよびTimeTracker FXのご運用環境について、以下の内容をご確認ください。

2.1 Web APIのインストール状況の確認

【確認内容】

Webサーバに「TimeTracker FX Web API」がインストールされていなければ、Web Editionを利用するWeb APIを実行できず、エラーが発生します。
ご利用のWebサーバに「TimeTracker FX Web API」がインストールされているか確認します。

IISマネージャを起動し、既定のWebサイト（Default Web Site以下）にWeb APIの仮想ディレクトリおよびアプリケーションが表示されているかをご確認ください。

【確認方法】

  1. Webサーバにて、IISマネージャを起動する。
  2. 既定のWebサイト（Default Web Site以下）にWeb APIの仮想ディレクトリおよびアプリケーションが表示されているかを確認する。

  • 「TimeTracker FX Web API」のインストール状況は、「プログラムの追加」からもご確認いただくことができます。
    「プログラムの追加」を表示し、表示される一覧より「TimeTracker FX Web API X.X.X」の有無をご確認ください。

【確認結果と対処方法】

  • Web APIがインストールされていない場合

WebサーバにWeb APIをインストールしてください。
Web APIのインストール方法につきましては、こちらをご参照ください。

  • Web APIがインストールされている場合

インストールされているWeb APIが利用できない状態である可能性がございます。
次項、「2.2 Web APIの利用環境の確認」の内容をご確認ください。

 

2.2 Web APIの利用環境の確認

【確認内容】

「TimeTracker FX Web API」がインストールされていても、実際にWeb APIのメソッドが利用できなければ情報が取得できず、エラーが発生します。
ご利用の環境で、Web APIが利用可能であるかを確認します。

【確認方法】

  1. Internet Explorerなどのブラウザを起動する。
  2. アドレスバーに「http://（マシン名）/（Web APIのWebサイト名）/xmlWebService.svc/system/accounts」を入力し、Enterキーで実行する。
  • Webサーバのマシン名およびWeb APIのWebサイト名は、ご利用環境に合わせてご変更ください

【確認結果と対処方法】

  • エラーが発生し、Web APIが実行されない場合

「Web APIが動作しない」の内容をご確認ください。

  • Web APIが実行され、結果が表示される場合

Web APIが正常に実行された場合、ログインアカウントの権限範囲内のアカウント情報がXML形式で出力されます。

Web APIの実行には問題がないため、ご利用いただいているPCの環境に問題がある可能性があります。
次項、「2.3 ブラウザのバージョン確認」の内容をご確認ください。

 

2.3 ブラウザのバージョン確認

【確認内容】

Web Editionを利用するブラウザが動作保証の対象外のバージョンの場合、Web Editionは正常に動作しません。
ご利用のブラウザのバージョンが、Web Editionの動作保証の対象のバージョンであるかどうかを確認します。

【確認方法】

  • Internet Explorerでのバージョン確認方法です。他のブラウザにつきましては、ブラウザの提供元にご確認ください
  1. Internet Explorerのメニューバーを表示する。
  2. ［ヘルプ］メニュー－［バージョン情報］を選択する。
  3. 表示されるダイアログにて、Internet Explorerのバージョンを確認する。

  1. 利用しているバージョンが、Web Editionの動作を保証する環境か確認する。
    なお、Web Editionが動作を保証する環境につきましては、こちらをご覧ください。

【確認結果と対処方法】

  • 動作保証の対象外のバージョンの場合

Web Editionが動作を保証するバージョンのブラウザをご用意ください。

  • 動作保証の対象のバージョンの場合

Web Editionのセットアップ内容に問題がある可能性がございます。
次項の「3.Web Editionのセットアップミス」の内容をご確認ください。

 

 

3.Web Editionのセットアップミス

TimeTracker FX Web Editionをご利用いただくためには、インストール（ファイルコピー）後に
IISやWeb Editionの設定ファイルなどいくつかの設定をご運用いただく環境に合わせて変更する必要があります。

これらの設定内容に誤りがある場合、Web Editionが正常に動作せず、エラーとなります。

3.1 Web Editionのインストール(ファイルコピー)内容の確認

【確認内容】

Webサーバでご利用のウイルスセキュリティソフトなどによっては、インストーラからのファイルのコピーが正常に完了せず、ファイルの欠損が発生する場合があります。
Web Editionのプログラムファイルが欠損した場合は、正常に動作せず、エラーが発生します。

プログラムファイルのコピーが正常に完了しているかどうかを確認します。

【確認方法】

  1. Web Editionのインストール先フォルダの容量を確認する。
  2. TimeTracker FXのインストーラに含まれる「Web Edition」フォルダの容量を確認する。
  3. インストール先フォルダとインストーラのファイル容量を比較する。

【確認結果と対処方法】

  • ファイル容量が異なる場合

Web Editionのプログラムファイルのコピーが正常に完了していない可能性がございます。
TimeTracker FXのインストーラより、再度、Web Editionのプログラムファイルをコピーしてください。

  • ファイル容量が同じ場合

インストールされたプログラムには問題がありません。
次項、「3.2 Web.configファイルへの追記内容の確認」の内容をご確認ください。

 

3.2 Web.configファイルへの追記内容の確認

【確認内容】

Web Editionでは、Web APIをSSL通信にて利用し、TimeTracker FXのデータベースから情報の操作を行います。
このため、Web APIにてSSL通信が利用できるようにWeb.configファイルの設定を編集します。

この編集内容に誤りがある場合、SSL通信によるWeb APIの接続に失敗し、エラーが発生します。

Web.configファイルの編集内容にて、追記箇所や記述内容に誤りがないかを確認します。

【確認方法】

  1. Web APIのインストーフォルダ直下にあるWeb.configファイルをテキストエディタで開きます。
  2. Web.configファイルへの編集内容および追記箇所を確認します。

<編集内容>

【確認結果と対処方法】

  • 編集内容および追記箇所に誤りがある場合

Web.configファイルの編集内容および追記箇所を修正し、IISマネージャよりIISのサービスを再起動して修正内容を反映します。

なお、インストール直後の初期設定ファイルをバックアップしている場合には、こちらのツールを利用してWeb.configファイルを再作成することをご推奨いたします。

  • 編集内容および追記箇所に誤りがない場合

次項、「3.3 app.jsonファイルの記載内容の確認」の内容をご確認ください。

 

3.3 app.jsonファイルの記載内容の確認

【確認内容】

Web Editionでは、「app.json」ファイルにて指定されたWeb APIを利用します。
このため、「app.json」ファイルの記載されたWebサーバのマシン名やWeb APIのWebサイト名などの情報に誤りがある場合、Web APIが利用できず、エラーが発生します。

「app.json」ファイルの記載内容に誤りがないかを確認します。

【確認方法】

  1. Web Editionのインストールフォルダ直下にあるapp.jsonファイルをテキストエディタで開きます。
  2. app.jsonファイルへの編集内容を確認します。

  • 末尾の「/（スラッシュ）」は必ず入力してください。

【確認結果と対処方法】

  • 記載内容に誤りがある場合

「app.json」ファイルの記載内容を修正し、Web Editionのログイン画面を表示するブラウザを再起動します。

  • 記載内容に誤りがない場合

次項、「3.4 IISの設定内容の確認」の内容をご確認ください。

 

3.4 IISの設定内容の確認

【確認内容】

Web Editionを利用するためには、IISにてSSL通信の設定やJSONファイルの利用設定を行う必要があります。
IISマネージャより、以下の内容について問題がないかを確認します。

【確認方法】

①サーバー証明書

Web Editionでは、SSLによる暗号化通信にてWeb APIを利用します。
このため、Web Editionを公開するWebサーバでは、SSLによる通信を利用するために「サーバー証明書」が必要となります。

以下の手順にてご用意いただいたサーバー証明書に有効期限切れなどの問題がないかを確認します。

【確認手順】

  1. IISマネージャを起動してWeb Editionをインストールした仮想ディレクトリを選択し、画面中央の［WEBSERVERホーム］から「サーバー証明書」をダブルクリックする。

  1. 画面中央の［サーバー証明書］に表示される一覧情報から用意したサーバー証明書を選択し、画面右側の［操作］の［表示］をクリックする。

  1. ［証明書］ダイアログに表示されるサーバー証明書の情報を確認し、有効期限切れなどの問題がないか確認する。

なお、サーバー証明書に何らかの問題がある場合には、再度、サーバー証明書を発行してください。

②httpsのバインド設定

用意した「サーバー証明書」は、SSL通信を利用するWeb APIのWebサイトにバインドとして登録する必要があります。

下記の手順にてWebサーバのサイトに、用意した証明書を使用したSSL通信のバインドが設定されているかどうかを確認します。

【確認手順】

  1. IISマネージャを起動して「Defalut Web Site」を選択し、画面右側の［操作］－［サイトの編集］の［バインド］をクリックする。

  1. ［サイト バインド］ダイアログに表示されるバインドの一覧から「https」を選択し、［編集］ボタンをクリックする。

  1. ［サイト バインドの編集］ダイアログを開き、［SSL証明書］に用意したサーバー証明書が指定されていることを確認する。

なお、SSL通信のバインドが設定されていない場合には、IISマネージャよりWeb APIのWebサイトにバインドを設定します。

③MIMEの登録

Web Editionのプログラムでは、JSON形式の情報を利用して情報の読み込みや設定を行います。
このため、Web Editionをインストールした仮想ディレクトリに「拡張子：.json」をMIMEとしての登録が必要となります。

以下の手順にて、Web Editionをインストールした仮想ディレクトリのMIMEの設定に「拡張子：.json」が登録されているかどうかを確認します。

【確認手順】

  1. IISマネージャを起動してWeb Editionをインストールした仮想ディレクトリを選択し、画面中央の［WebEditionホーム］から「MIMEの種類」をダブルクリックする。

  1. 画面中央の［MIMEの種類］に表示される一覧情報に、「拡張子：.json」が登録されているか確認する。

なお、MIMEに「拡張子：.json」が登録されていない場合は、［追加］ボタンから「拡張子：.json」をMIMEに追加します。

【確認結果と対処方法】

  • IISの設定に問題があった場合

IISを再起動し、修正した設定内容を反映します。

  • IISの設定に問題がない場合

次項、「3.5 アクセス権の確認」の内容をご確認ください。

 

3.5 アクセス権の確認

【確認内容】

Web APIおよびWeb Editionでは、IISで公開するWeb APIのWebサイトに以下のアクセス権が必要となります。

No. 機能 アカウントorグループ 権限 備考
1 Web API Usersグループ 書き込み・読み込み権限 ログイン認証時に発行する
アクセストークンの保存に利用します。

すでにIISを他のアプリケーションで利用しており、既定のWebサイトのアクセス権を独自に設定している場合、
設定状況によっては、上記権限が不足する可能性がございます。

Web AIPのWebサイトのアクセス権にて、設定された権限の状態を確認します。

【確認方法】

  1. Webサーバにて、IISマネージャを起動する。
  2. ［接続］より「Default Web Site」直下にあるWeb APIのWebサイトのアクセス権の設定を確認する。

【確認結果と対処方法】

  • 権限が不足している場合

すでにIISを他のアプリケーションで利用しており、既定のWebサイトのアクセス権を独自に設定している場合など、
設定状況によっては、必要となる権限の設定が不足する可能性がございます。

上記のインストール時に設定されるデフォルトのアクセス権設定と比較し、不足しているアクセス権の追加または設定を実施ください。

  • 権限は不足していない場合

次項、「3.6 IISへの最新データの読み込み確認」の内容をご確認ください。

 

3.6 IISへの最新データの読み込み確認

【確認内容】

IISは、Windowsのサービスとして起動し、起動時に設定された情報をもとに動作します。
このため、各種設定後にIISを再起動し、編集した設定内容をIISに適用する必要がございます。

【確認結果と対処方法】

  • 各種設定後、IISを再起動していない場合

各種設定後、IISの再起動を実施していない場合には、IISマネージャより、IISの再起動を実施してください。

  • IISを再起動している場合
