TimeTracker FXはサーバマシンのデータベースに接続し、実績やプロジェクト計画の情報を蓄積します。
TimeTracker FXがデータベースに接続できなくなった場合は、ネットワーク環境などの設定が正しいかご確認ください。

症状

以下のような状況の場合に、TimeTracker FXのログイン画面で、ログイン名とパスワードを入力しても、エラー画面を表示し、TimeTracker FXが起動しない。

  • TimeTracker FXをインストールして、初めてサーバマシンにアクセスする場合
  • サーバマシンの交換をして、初めてサーバマシンにアクセスする場合
  • サーバマシンの場所が移動して、初めてサーバマシンにアクセスする場合

 

原因と対策

本現象が起こる可能性の高い原因を列挙します。
各原因が該当しないかご確認ください。
また、各原因についての対応方法は以降に記載しております。

  • 動作対象外のバージョンを使用している
  • サーバマシンのマシン名からIPアドレスを特定できない環境でTimeTracker FXを使用している
  • SQL Serverの通信プロトコルでTCP/IPを使用できていないため、TimeTracker FXがSQL Serverに接続できない
  • SQL Server Expressのリモート接続を有効にしていますか
  • サーバマシンのファイアウォールで、SQL Serverとの通信を遮断しているため接続できない
  • SQL Server Browserサービスが停止している
  • サーバ設定ファイルが正しく参照できていないか破損している
 

これらの原因が該当しない場合は、お問い合わせに必要な情報、およびご利用中のサーバ・クライアントの環境、TimeTracker FXで参照しているサーバ設定ファイルを添えて、こちらまでご連絡ください。

 

動作対象外のバージョンを使用している

【原因】

ご利用環境が、動作環境と一致していない場合、TimeTracker FXを使用できません。
こちらより動作環境の詳細をご確認ください。

【対策】

動作環境と一致するよう、ご利用環境を構築してください。

対策を行っても状況が改善されない場合は、次の原因をご確認ください。

 

サーバマシンのマシン名からIPアドレスを特定できない環境でTimeTracker FXを使用している

【原因】

TimeTracker FXは、サーバマシンのマシン名を使用して、サーバマシンとの接続を行います。
そのため、名前の解決で、サーバマシンのIPアドレスを特定できない環境では、接続先が見つからず、TimeTracker FXを使用できません。

【対策】

以下のいずれかの方法で、サーバマシンのIPアドレスを特定できる環境にしてください。

  • DNSの設定を修正し、サーバマシンのマシン名からIPアドレスを特定できるようにする
    • DNSの設定は、ご利用のDNSによって異なります。
      詳細はご利用のDNSのマニュアルをご覧ください。
  • IPアドレスを使用したサーバ設定ファイルを作成して利用する

以下に各バージョンでの作成手順を紹介します。
ご利用のバージョンの作成手順を行ってください。

［TimeTracker FX 2.5以降］

Server Manager ヘルプの［サーバ設定ファイルを作成する］をご覧ください。

［TimeTracker FX 2.0.x］

  1. Windows［スタート］－［すべてのプログラム］－［TimeTracker FX］－［管理ツール］－［サーバ設定ファイル作成ツール］を選択します。
  2. ［サーバ設定ファイル作成ツール］ダイアログで、［更新］をクリックします。
  3. ［サーバ名］のマシン名をIPアドレスに変更します。
  4. ［ログイン名］、［パスワード］を設定し、［データベース名］をリストから選択します。
  5. ［参照］をクリックし、表示する［フォルダの参照］ダイアログで、サーバ設定ファイルを出力するフォルダを選択します。
  6. ［作成］をクリックし、サーバ設定ファイルを作成します。

対策を行っても状況が改善されない場合は、次の原因をご確認ください。

 

SQL Serverの通信プロトコルでTCP/IPを使用できていないため、TimeTracker FXがSQL Serverに接続できない

【原因】

SQL Serverの通信プロトコルでTCP/IPを使用できていないためSQL Serverに接続できない可能性があります。

【対策】

SQL Serverの通信プロトコルをTCP/IPのみ有効にしてください。
SQL Serverのバージョンによって、設定方法が異なります。
各バージョンの設定方法は、以下の手順をご覧ください。

［SQL Server 2000］

  1. Windows［スタート］－［ファイル名を指定して実行］を選択します。
  2. svrnetcn と入力します。
  3. 表示されるダイアログで、［有効になっているプロトコル］が「TCP/IP」のみにします。
  4. ［OK］をクリックします。
  5. SQL Serverのサービスを再起動します。

［SQL Server 2005および、SQL Server Express］

  1. Windows［スタート］－［Microsoft SQL Server 2005］－［構成ツール］－［SQL Server 構成マネージャー］を選択します。
  2. ［SQL Server 2005 ネットワークの構成］－［TimeTracker FXのインスタンス名のプロトコル］を選択します。
    SQL Server Expressをオートランからインストールした場合は、［TIMETRACKERFXのプロトコル］を選択します。
  3. 表示されるプロトコルから、TCP/IPを［有効］にします。
  4. 共有メモリ、名前付きパイプ、VIAを［無効］にします。
  5. ［SQL Server 2005 のサービス］を選択します。
  6. SQL Server (TimeTracker FXのインスタンス名)を再起動します。
    SQL Server Expressをオートランからインストールした場合は、SQL Server (TIMETRACKERFX)を再起動します。

対策を行っても状況が改善されない場合は、次の原因をご確認ください。

 

SQL Server Expressのリモート接続を有効にしていますか

【原因】

SQL Server Expressは既定でネットワーク接続を無効としています。

【対策】

SQL Server Expressのリモート接続を有効にする方法については、こちらをご覧ください。

 

サーバマシンのファイアウォールで、SQL Serverの通信を遮断しているため接続できない

【原因】

サーバマシンがWindows XP SP2以降、またはWindows Vista以降の場合、Windowsファイアウォールが有効となっていてSQL Serverとの通信を遮断している可能性があります。

【対策】

Windowsファイアウォールを有効にした状態で、SQL Serverの通信を許可する方法については、こちらをご覧ください。

 

SQL Server Browserサービスが停止している

【原因】

TimeTracker FXで使用するデータベースが［名前付きインスタンス］で作成されている場合、SQL Server Browserサービスが停止していることでクライアントPCからのデータベース接続に失敗する可能性があります。

【対策】

SQL Server 構成マネージャより、SQL Server Browserサービスを開始してください。
SQL Server Browserサービスを開始する方法については、こちらをご覧ください。

 

サーバ設定ファイルが正しく参照できていないか破損している

【原因】

サーバ設定ファイル（TimeTracker.config）とは、TimeTracker FXがデータベースに接続するための情報を格納したファイルです。
何らかの原因により、サーバ設定ファイルが正しく参照できていないか破損している可能性があります。

【対策】

TimeTracker FXを正常にご利用いただけるメンバがお使いのサーバ設定ファイルをご利用ください。
データベースに接続できましたら、そのままTimeTracker FXをご利用ください。
