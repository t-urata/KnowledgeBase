ダッシュボードやアナリストのデータが表示されません

更新日:

2016/3/10

対象バージョン:

3.0以降

対象エディション:

Professional / Standard

 

ダッシュボードやアナリストのデータは、サーバタスクというサービスが集計します。
そのため、サーバタスクが正常に動作しない場合、ダッシュボードやアナリストのデータが表示されません。

症状

具体的な症状として、以下の症状があります。

［ダッシュボード］

  • プロジェクトモニタにデータが表示されない
  • プロジェクト推移モニタにデータが表示されない
  • プロジェクト推移モニタで、対象ノードを選択できない

［アナリスト］

  • プロジェクトサマリでデータが表示されない
  • ガントチャートにプロジェクトやタスクが表示されない
  • ガントチャートの推移モニタが表示されない
  • マイルストーンサマリでデータが表示されない
  • リソースサマリで実績が表示されない
  • リソース負荷で計画工数・実績工数が表示されない

原因と対策

本現象が起こる可能性の高い原因から順にケース1～9として説明いたします、ケース1から順にご確認ください。

［ケース1］

【よくあるケース】

サーバマシンをメンテナンスするために、サーバのシャットダウンや再起動を行った。

【原因】

サーバマシン起動時にTimeTrackerサーバタスクのサービスがSQL Serverのサービスより早く開始し、TimeTrackerサーバタスクのサービスが正しく起動できない。

原因の詳細については、こちらをご覧ください。

【対策と確認】

ご利用されているバージョンに対応する、以下の手順を行ってください

  1. Server Managerを開きます。
  2. ［サーバタスクの管理］の［サーバタスクを停止する］をクリックします。
  3. ［サーバタスクを開始する］をクリックします。
  4. ［サーバタスクの状態］が［稼働中］となっていることを確認します。
    • 本回避方法で対応できた場合は、次回サーバマシンを再起動したときに再発する可能性がございます。
      サーバマシンを再起動するときに、サーバタスクが正しく起動できたかご確認ください。

ケース1で解決しない場合は、ケース2をご確認ください。

［ケース2］

【よくあるケース】

サーバマシンを使用しない間、シャットダウンしていた。

【原因】

サーバタスクの実行予定時刻に、サーバマシンがシャットダウンしていた。

【対策と確認】

  1. Administratorを開きます。
  2. ［サーバタスクの設定］を選択します。
  3. ［サーバタスク一覧］の［推移データの作成］及び［プロジェクト情報の更新］について、［曜日］欄と［時刻］欄を変更します。
  4. ［データベースに反映］をクリックします。
  5. 翌週以降、［前回実行日時］・［次回実行日時］欄がスケジュール通り変更されていることを確認します。
    • サーバマシンのシャットダウンと、サーバタスクの実行時間が重複しなければ、この原因では再発しません。

ケース2で解決しない場合は、ケース3をご確認ください。

［ケース3］

【よくあるケース】

データベースをバックアップするために、データベースを停止した。

【原因】

データベースが停止している。

【対策と確認】

  1. Windows［スタート］-［ファイル名を指定して実行］を選択します。
  2. ［ファイル名を指定して実行］ダイアログに services.msc と入力します。
  3. ［OK］をクリックし、［サービス］ダイアログを表示します。
  4. SQL Serverのサービスを右クリックします。
  5. 表示するポップアップメニューの［開始］を選択します。
  6. SQL Serverのサービスの［状態］が［開始］となっていることを確認します。
    • SQL Serverのサービスは、バージョンやインスタンスの設定によって、サービス名が異なります。
      サービス名については、 こちらをご覧ください

      ［SQL Server 2000］
      ・既定のインスタンスの場合：MSSQLSERVER・名前付きインスタンスの場合：MSSQL$インスタンス名

      ［SQL Server Express］
      ・既定のインスタンスの場合：SQL Server (MSSQLSERVER)

      ［SQL Server 2005］
      ・既定のインスタンスの場合：SQL Server (MSSQLSERVER)
      ・名前付きインスタンスの場合：SQL Server (インスタンス名)

    • データベースのサービスを停止しなければ、この原因では再発しません。

 

ケース3で解決しない場合は、ケース4をご確認ください。

［ケース7］

【よくあるケース】

ケース7で解決しない場合は、ケース8をご確認ください。
