問題の発生するバージョン

TimeTracker FX 2.5.0

 

問題が修正されるバージョン

未定

 

内容

TimeTracker FX 2.0.xで作成したデータベースをコピー後、コピー元となったデータベースが復元できなくなります。

  1. TimeTracker FX 2.0.xで作成したデータベースを、Server Managerに登録する。

  2. 登録したデータベースを、Server Managerの［データベースをコピーする］機能でコピーする。

  3. コピー元のデータベースを、Server Managerの［手動でバックアップする］機能でバックアップする。

  4. Server Managerの［データベースを復元する］機能でコピー元のデータベースを復元する。

  5. 復元に失敗する。

 

回避方法

以下の手順を行ってください。

  1. Server Managerを起動します。

  2. データベースの管理で復元に失敗したデータベースを選択します。

  3. 復元に失敗したデータベースの管理画面で［データベースの登録を解除する］をクリックします。

  4. データベースの登録を解除した後、Server Managerのホーム画面に戻ります。

  5. ［既存のデータファイルをサーバに登録する］クリックし、復元に失敗したデータベースを再度登録し直します。
