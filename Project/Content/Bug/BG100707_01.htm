SQL Server 2008以降で定期バックアップを実行するには

更新日:

2016/7/26

対象バージョン:

2.7 ~ 3.8

対象エディション:

Professional / Standard

現象

SQL Server 2008以降のSQL Serverを利用した環境で、TimeTracker FX 同梱のデータベース管理ツールServer Managerの定期バックアップを設定しても、定期バックアップが実行されません。

原因と対策

SQL Server 2008 以降では、SQL分散管理オブジェクト(SQL-DMO)が標準のインストールに含まれません。
Server Managerの定期バックアップは、SQL-DMOを利用しています。
このため、定期バックアップを実行するにはSQL-DMOを追加でインストールする必要があります。
SQL-DMOの入手方法やインストール手順の詳細は[SQL-DMO_Install.pdf]をご覧ください。

[SQL-DMO_Install.pdf]の手順で定期バックアップが実行されない場合は、SQL Serverのメンテナンスプランを利用する方法があります。
手順の詳細は[SQLServer_MaintenancePlan_Backup.pdf]をご覧ください。
この手順でServer Managerの定期バックアップが実行されない場合は、Microsoft社までお問い合わせください。
