問題の発生するバージョン

TimeTracker FX 2.0.0

TimeTracker FX 2.0.1

 

問題が修正されるバージョン

TimeTracker FX 2.0.2

 

内容

以下の条件を満たす場合、サーバタスクの［リソース負荷の更新］が失敗して、リソース負荷のデータが表示されないことがあります。

  • SQL Server 2005 を利用している

  • 互換性レベルを［SQL Server 2005 (90)］に設定している

  • SQL Server 2005の既定の設定は、［SQL Server 2005 (90)］です。

 

回避方法

TimeTracker FX 2.0.2をご利用ください。

こちらの方法でも回避できます。

以下の手順で、データベースの互換性レベルを［SQL Server 2000 (80)］に設定します。

  1. SQL Server Management Studioを起動します。

  2. ［オブジェクトエクスプローラ］から［サーバマシン名］-［データベース］-［TimeTracker FXのデータベース］を選択します。

  1. ［TimeTracker FXのデータベース］を右クリックして、［プロパティ］をクリックします。

  2. ［オプション］をクリックします。

  3. ［互換性レベル］を［SQL Server 2000 (80)］に設定して、［OK］ボタンをクリックします。

  1. SQL Server Management Studioを終了します。

 

再集計について

互換レベル［SQL Server 2005 (90)］で動作していた間のリソース負荷データは、データベースの設定を変更しても集計されません。

以下のツールを実行して、再集計を行ってください。

  1. ［リソース負荷データ更新ツール］をダウンロードします。

  2. ダウンロードした［リソース負荷データ更新ツール］を任意のフォルダへ展開します。

  • ［リソース負荷データ更新ツール］は、クライアントPCで実行できます。

  1. リソース負荷データ更新ツール.xlsを、 マクロを有効にした状態で開きます。

    【Excelマクロを有効にする方法】

    1. Excelを起動します。

    2. Excelの［ツール］メニューから［マクロ］-［セキュリティ］をクリックします。

    3. セキュリティダイアログで［中］を選択します。

    4. ［OK］ボタンをクリックします。

    5. Excelを終了します。

    6. リソース負荷データ更新ツール.xlsを開きます。

  2. リソース負荷データ更新ツール.xlsに記載されている手順の通りに操作を行います。
