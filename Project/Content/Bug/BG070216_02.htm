問題が修正されるバージョン

TimeTracker FX 2.0.2

 

内容

フィールド値の決定方法に、0で除算するなどの計算式を設定すると［情報］フィールドに警告が発生します。

以下の条件1または条件2のいずれかを満たす場合、計算式を修正してもプロジェクトを開きなおすと、対象フィールドに問題がなくても警告が発生します。

 

【条件1】

  • ノードのカスタムフィールド（数値1～30）の［フィールド値の決定方法］に、［100/0］などエラーとなる計算式を設定している

  • エラーとなる計算式を設定したノードで、他の全フィールドに計算式を設定していない

 

【条件2】

  • ［デフォルトの決定方法］に、［100/0］などエラーとなる計算式を設定している

  • デフォルトの計算式の結果がエラーとなったノードで、エラーの発生したフィールドを含め全フィールドに計算式を設定していない

 

【条件1】の具体例は次の通りです。

  1. 任意のノードの［カスタムフィールド（数値1～30）］を選択します。

  2. ［計算式バー］に［100/0］を入力します。

  3. ［情報］フィールドに警告アイコンが表示されます。

  4. プロジェクト計画を保存します。

  5. 対象のフィールドを右クリックして、［フィールド値のクリア］を選択します。

  6. 計算式が削除され、警告アイコンが消えます。

  7. プロジェクトを保存して閉じます。

  8. プロジェクトを開くと、［情報］フィールドに警告アイコンが表示されます。

 

回避方法

TimeTracker FX 2.0.2をご利用下さい。

こちらの方法でも回避できます。

条件演算を使用して、エラーを回避してください。

  • 回避例

PlannedTime <> 0 ? (ActualTime / PlannedTime * 100) :0

  • 計画工数が0でなければ、実績工数÷計画工数×100、0であれば、0となります。
