問題の発生するバージョン

TimeTracker FX 2.0.0 Standard Edition

 

問題が修正されるバージョン

TimeTracker FX 2.0.1 Standard Edition

 

内容

プランナーやオートメーションAPIで、Microsoft Projectからプロジェクトファイルをインポートするとエラーが発生する場合があります。

  • TimeTracker FX Professional Editionでは、この問題は発生しません。

 

このエラーは、以下の条件を満たす場合に発生する場合があります。

  • TimeTracker FX Standard Editionを利用している

  • Microsoft Projectのプロジェクトに、TimeTracker FXで登録しているアカウントのリソース割り当てがある

  • Microsoft Projectのプロジェクトに、リソースごとの計画工数が1以上に設定されている

  • TimeTracker FXのインポート先プロジェクトに、リソースとして割り当てるメンバのコストに0より大きな値が設定されている

 

回避方法

エラーが発生するまでのノードはインポートされています。

インポートされたデータを削除または、破棄をして、以下の手順で改めてインポートを行ってください。

  1. Microsoft Projectのプロジェクトに割り当てているリソースを、すべてTimeTracker FXのアカウントとしてAdministratorで登録します。

  2. Microsoft Projectのプロジェクトに割り当てているリソースを、TimeTracker FXのインポート先プロジェクトにメンバとして追加します。

  3. リソースとして割り当てるメンバのコスト単価をメモしておき、コスト単価を0に設定します。

  4. Microsoft Projectからインポートを行います。

  5. 3で変更したコスト単価を元に戻します。

  6. Administratorの［アカウントの編集］でコスト単価が設定されており、インポート先プロジェクトで変更していない場合は、

［すべてシステム定義に戻す］ボタンをクリックすると、元のコスト単価に戻すことができます。
