問題の発生するバージョン

TimeTracker FX 2.5.0

 

問題が修正されるバージョン

未定

 

内容

SQL Server 2005で、データベースを格納する既定のフォルダを、ルートフォルダ（例えば C:\ ）に設定すると、Server Managerで既存のデータファイルをサーバに登録しても、データファイルが指定した場所に置けません。

  1. SQL Server Management StudioまたはSQL Server Management Studio Expressで既定フォルダをルートフォルダに指定する。

  2. Server Managerで［既存のデータファイルをサーバに登録する］を選択する。

  3. データベース登録ウィザードでデータファイルを登録時に［既定のデータフォルダにコピーして登録する］を選択する。

  4. OSによって、以下の状態のいずれかとなる。

  • ［次へ］をクリックするとエラーが発生する

  • そのまま登録できるが、データベースはServer Managerのプログラムが格納されているフォルダに置かれる

 

回避方法

SQL Server Management StudioまたはSQL Server Management Studio Expressで既定フォルダをルートフォルダ以外に設定してください。
