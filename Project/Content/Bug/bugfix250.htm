TimeTracker FX 2.5で修正された問題点や不具合です。

 

データベース作成ウィザード

  • データベース作成ウィザードの処理完了ダイアログのメッセージが編集できる

 

プランナー

  • クライアントPCで地域オプションを変更すると、プロジェクト保存時にシステムエラーが発生する場合がある

  • ［表示するベースラインの変更］ダイアログのベースライン選択リストに空行を表示する場合がある

  • プロジェクトの設定で、開始日または終了日を削除するとシステムエラーが発生する

  • プロジェクト計画をフィルタ中に、プロジェクト設定を更新するとプロジェクト名の横に［フィルタ中］と表示される

  • プランナーの［開始日］･［終了日］フィールドに、時間も入力できる

  • ［進捗率］フィールドに値を入力するとき、１文字目が入力されない場合がある

  • 親ノードと子ノードがお互いのフィールド値を参照するとき、システムエラーが発生する場合がある

 

タイムシート

  • Outlookから予定をインポートするとき、予定がインポートされない場合がある

  • 実績レポートで集計区分をタスクで表示すると、正しくグループ化されない場合がある

  • マイタスクで名前編集中にタスクを削除するとシステムエラーが発生する

  • タイムシートのタイムラインで再読み込みを行うと、実績が表示されなくなる

 

アナリスト

  • アナリストのマイルストーンサマリでタイムラインがグループ化できない場合がある

 

ダッシュボード

  • プロジェクト名を変更しても、ダッシュボードの推移モニタに反映されない

 

ピボット分析

  • プロジェクトカテゴリ名を変更しても、ピボット分析のビューに反映されない

 

オートメーション

  • オートメーションで非表示のままログインをすると、表示設定を変更してもタイムシートは非表示のままになる

  • オートメーションで、アプリケーションを非表示のまま終了するとプロジェクトが編集中になる

  • ノード追加直後にGetWBSNodeOutlineNumberでアウトライン番号が取得できない

 

Administrator

  • プロジェクトカテゴリを編集中に強制終了すると、Administratorが起動できなくなる

  • Administratorでサーバタスクの処理時間が正しく表示されない

 

全般・その他

  • Windows Server 2003 SP1にTimeTracker FXをインストールできない場合がある

  • 実績のエクスポートで、開始日または終了日を削除するとシステムエラーが発生する

  • グリッド表示で使用するフィルタの［フィルタ条件］ダイアログに誤字がある

  • 1プロジェクト内に同一のタスクコードがある場合、サーバタスクの［プロジェクト情報の更新］が失敗する場合がある

  • プロジェクトのリソース負荷が正しく集計されない場合がある

  • サーバマシン起動時にサーバタスクが正しく開始しない場合がある

  • イベントログに書き込みできないと、サーバタスクが起動しない

  • サーバタスクの処理中にデータベースが停止した場合、処理中のまま残る

  • サーバタスクの処理中に編集中のプロジェクトが多数存在するとサーバタスクの処理が増加する
