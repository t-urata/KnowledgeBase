問題の発生するバージョン

TimeTracker FX 2.0.0以降

 

問題が修正されるバージョン

TimeTracker FX 2.5.0

 

内容

サーバマシン起動時に、サーバタスクが正しく起動しないことがあります。

 

【通常の起動】

  1. サーバマシンが起動します。

  2. SQL Serverが起動します。

  3. TimeTracker FXのデータベースが開始します。

  4. サーバタスクが開始します。

 

【問題がある場合の起動】

  1. サーバマシンが起動します。

  2. SQL Serverが起動します。

  3. サーバタスクが開始します。
    TimeTracker FXのデータベースが起動しておらず必要な情報が取得できません。

  4. TimeTracker FXのデータベースが開始します。

  5. サーバタスクは［開始］状態ですが処理を行いません。

 

サーバタスクが正常に開始したことを確認するには、こちらをご覧ください。

 

回避方法

以下の手順で操作します。

  1. サーバマシンを起動したときに、サーバタスクが正しく開始したことを確認します。手順はこちらをご覧ください。
    正しく開始している場合は、以降の操作は不要です。

  2. Administratorを起動します。

  3. ［その他］グループの［サーバタスク］をクリックします。

  4. ［停止］ボタンをクリックします。

  5. ［開始］ボタンをクリックします。

  6. サーバタスクが正常に起動したことを確認します。手順はこちらをご覧ください。
