アカウント一覧で、コスト単価が実際の値に関わらず「0」と表示される場合がある

更新日:

2014/12/5

対象バージョン:

3.5以降

対象エディション:

Professional / Standard

現象

Administratorツールに管理用アカウント「Administrator」でログインした際に、[アカウントの編集]ビュー上の[コスト単価]の値が、実際の値に関わらずすべて「0」と表示される場合があります。

  • 管理用アカウント「Administrator」以外のアカウントでは本現象は発生しません。
    したがって、Administratorツールへのログインにアカウント「Administrator」を使用しない場合は、本コンテンツの問題は発生しません。

正常なケース(コスト単価の値が確認できる)

現象が発生するケース(コスト単価が「0」で表示される)

影響範囲

この状態でもコスト単価の編集・更新は可能ですが、保存(データベースに反映)した後は表示が再び「0」となります。
(編集内容は保存されています。)
また、コスト単価以外の列を更新して保存した場合、コスト単価は「0」で更新されてしまいます。

発生条件

既定で用意されている役割である「管理者」(または「DB管理者」)について、以下のいずれかの条件のいずれかを満たす場合に本現象が発生します。

  • [無効]がチェックされている
  • [DB管理]がチェックされていない

なお、既定の役割「管理者」の名前が変更されている場合は、以下の方法で特定できます。

  1. Administratorを起動します。(既に開いている場合は再起動します。)
  2. [役割の定義]ビューを表示します。
  3. 「無効な役割を非表示」のチェックを外します。
  4. 一番上に表示される役割が該当します。

回避方法

以下の設定を行うことにより、コスト単価を正しく表示することができます。

  • 役割「管理者」について、[DB管理]をチェックし、[無効]のチェックを外す

なお、役割「管理者」(以下「既定の役割」)の設定を変更して別の役割として使用している場合は、以下の手順により対応可能です。

  1. 新規に役割を追加します。
  2. 追加した役割の権限を、「既定の役割」の現在の設定内容と同一にします。
  3. [データベースに反映]をクリックします。
  4. [アカウントの編集]ビューで、役割が「既定の役割」に設定されているアカウントの役割を、新規に追加した役割に変更します。
  5. [データベースに反映]をクリックします。
  6. 「既定の役割」の設定について[DB管理]をチェックし、[無効]のチェックを外します。
  7. [データベースに反映]をクリックします。
