サーバマシンのWindowsファイアウォールを有効にした状態で、TimeTracker FXを利用する方法を説明します。

  • TimeTracker FXが利用するインスタンス以外に他のインスタンスが存在する場合は、この設定をできない可能性があります。

 

このトピックの対象

  • Windows Server 2008以降
  • Windows Vista以降　※評価版のみ
  • Windows XP SP2以降　※評価版のみ
  • 上記以外のOSでWindowsファイアウォールを有効に設定した場合

 

事前に調べておくこと

Windowsファイアウォールで、以下のプログラムの通信を許可します。プログラムのファイルパスをメモなどに控えてください。

  • sqlservr.exe
  • sqlbrowser.exe
    • sqlbrowser.exeは、SQL Server 2005以降で追加されたプログラムです。

【ファイルパスの確認方法】

  1. Windows［スタート］－［すべてのプログラム］－［アクセサリ］－［ファイル名を指定して実行］を選択します。
    ※Windows XPをご利用の場合、 Windows［スタート］－［ファイル名を指定して実行］を選択します。
  2. ［ファイル名を指定して実行］ダイアログの［名前］欄に　services.msc　を入力し、［OK］をクリックします。
  3. サービスの一覧から、TimeTracker FXが利用しているSQL Serverのインスタンス名を右クリックします。
    評価版に同梱のSQL Server Expressをご利用の場合は、SQL Server (TIMETRACKERFX)となります。
  4. 表示されるメニューから［プロパティ］を選択します。
  5. ［実行ファイルのパス］欄の、"　" で囲まれたパスが、sqlservr.exeの場所となります。
  6. 同様に、SQL Server Browserのプロパティを表示して、パスを確認してください。
    • SQL Server 2000の場合、手順6.は不要です。

Windowsファイアウォールの設定をする

Windowsファイアウォールで、プログラムを例外に指定し、通信を遮断しないように設定します。

【手順（Windows Server 2008以降またはWindows Vista以降）】

  1. Windows［スタート］から、［コントロールパネル］を選択します。
  2. ［コントロールパネル］ダイアログから［システムとセキュリティ］をクリックします。
  3. ［システムとセキュリティ］ダイアログから［Windows ファイアウォールによるプログラムの許可］をクリックします。
  4. ［設定の変更］ボタンをクリックします。
  5. ［別プログラムの許可］ボタンをクリックし、［プロジェクトの追加］ダイアログを表示します。
  6. ［参照］をクリックして、表示されるダイアログの［ファイル名］欄に事前に調べておいた sqlservr.exe のファイルパスを貼り付け、
    ［開く］ボタンをクリックします。
  7. ［追加］ボタンをクリックします。
  8. 許可されたプロジェクトおよび機能の一覧に追加したプログラムが表示されるため、ホーム/社内の欄にチェックを入れます。
  9. ［sqlbrowser.exe］に対して、手順4.から手順8.を行います。
  10. ［OK］をクリックし、［Windowsファイアウォール］ダイアログを閉じます。

【手順(上記以外のOS)】

  1. Windows［スタート］から、［コントロールパネル］を選択します。
  2. ［コントロールパネル］ダイアログから［Windows ファイアウォール］をクリックします。
  3. ［Windows ファイアウォール］ダイアログで、［例外］ タブをクリックし、［プログラムの追加］をクリックします。
  4. ［参照］をクリックして、表示されるダイアログの［ファイル名］欄に事前に調べておいた sqlservr.exe のファイルパスを貼り付け、
    ［開く］ボタンをクリックします。
  5. ［OK］をクリックし、［プログラムの追加］ダイアログを閉じます。
  6. sqlbrowser.exeに対して、手順3.から手順5.を行います。
    • SQL Server 2000の場合、手順6.は不要です。
  7. ［OK］をクリックし、［Windowsファイアウォール］ダイアログを閉じます。
