• 本手順は、TimeTracker FX サポートチームより案内があった場合に行ってください。

  • 既定のインスタンスと名前付きインスタンスが共存している環境では、本手順を行わないでください。

 

内容

サーバマシンのWindowsファイアウォールを有効にした状態で、SQL Serverの通信ポートを許可する方法を説明します。

 

前提条件

名前付きインスタンスを利用している場合、事前にTCPポートを固定する必要があります。

本手順では、名前付きインスタンスのTCPポートを「1433」に固定している場合を例にしています。

  • 利用する通信ポートについてはこちらをご覧ください。

  • 名前付きインスタンスを利用している環境で、通信ポートを固定したい場合は以下のトピックをご覧ください。

  • SQL Server 2005以降、またはSQL Server 2005以降のExpressを利用されている場合はこちら

  1. Windowsスタートメニューの［コントロールパネル］を開きます。

  2. ［セキュリティセンター］の［Windowsファイアウォール］をクリックします。

  3. ［Windowsファイアウォール］ダイアログが開きます。

  4. ［例外を許可しない］のチェックをはずします。

  5. ［例外］タブをクリックします。

  6. ［ポートの追加］ボタンをクリックします。

  7. ［ポートの追加ダイアログ］にTCPポートの設定を行います。

    • ［名前］･･･ms-sql-s（任意）

    • ［ポート番号］･･･1433

    • ［TCP/UDP］･･･TCP

  1. ［OK］ボタンをクリックして、［ポートの追加ダイアログ］ダイアログを閉じます。

  2. ［ポートの追加ダイアログ］にUDPポートの設定を行います。

    • ［名前］･･･ms-sql-m（任意）

    • ［ポート番号］･･･1434

    • ［TCP/UDP］･･･UDP

  • 既定のインスタンスで動作している場合は必要ありません。
