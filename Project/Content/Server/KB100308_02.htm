このトピックの対象

SQL Server 2005以降、およびSQL Server 2005以降のExpress

 

内容

名前付きインスタンスを利用している場合に、通信ポートを固定する方法を説明します。

  • 既定のインスタンスと名前付きインスタンスが共存している環境では、本手順を行わないでください。

  • 本手順は、既定のインスタンスで動作している場合は必要ありません。

  • TimeTracker FX 2.5以降の評価版でautorunからSQL Server Expressをインストールした場合は、必ず行ってください。

  • 本手順はSQL Server 2005で説明しますが、SQL Server 2008以降の場合も手順は同様です。

 

設定手順

 

  1. Windowsスタートメニューの［すべてのプログラム］－［Microsoft SQL Server 2005］－［構成ツール］－［SQL Server 構成マネージャ］を選択します。

  2. ［SQL Server 構成マネージャ］－［SQL Server 2005 ネットワークの構成］－［（インスタンス名）のプロトコル］を選択します。
    インスタンス名は、TimeTracker FXのデータベースが接続されたインスタンスの名前になります。

  • TimeTracker FX 2.5以降の評価版でautorunからSQL Server Expressをインストールした場合は、インスタンス名は「TIMETRACKERFX」です。

  1. ［プロトコル名］の［TCP/IP］をクリックします。

  2. ツールバーの［プロパティ］ボタンをクリックします。

  1. ［TCP/IPのプロパティ］ダイアログで［IP アドレス］タブを選択します。

  2. ［IP1］の［TCPポート］に、［1433］と入力します。

  3. ［IP1］の［TCP動的ポート］の値を削除します。

  1. 各ダイアログにて、［OK］ボタンをクリックして、画面を閉じます。

  2. ［SQL Server 構成マネージャ］－［SQL Server 2005 のサービス］を選択します。

  3. ［SQL Server （インスタンス名）］を選択し、ツールバーの［サービスの再起動］ボタンをクリックします。
    インスタンス名は、TimeTracker FXのデータベースが接続されたインスタンスの名前になります。

  • TimeTracker FX 2.5以降の評価版でautorunからSQL Server Expressをインストールした場合は、インスタンス名は「TIMETRACKERFX」です。
