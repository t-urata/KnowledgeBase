Windows 8やWindows Server 2012で.NET Framework 2.0を利用できるようにする

更新日:

2016/6/26

対象バージョン:

3.7, 3.8

対象エディション:

Professional / Standard

 

TimeTracker FXには.NET Framework 2.0が必要ですが、Windows 8やWindows Server 2012では.NET Framework 2.0が利用できない状態になっていることがあります。
このトピックでは、これらのOSで.NET Framework 2.0を利用できる状態にするための方法を紹介します。

対象OS

本トピックが対象とするOSは以下のとおりです。

  • サーバーOS： Windows Server 2012, 2012 R2
  • クライアントOS： Windows 8, 8.1

設定方法

対象OSで.NET Framework 2.0を利用できる状態にするには、以下の2つの方法があります。

  • コントロールパネルから.NET Framework 2.0を利用できるようにする
    通常はこちらの方法を実施してください。
    マウス操作のみで設定でき、特殊な操作は不要です。
  • コマンドプロンプトから.NET Framework 2.0を利用できるようにする
    Microsoft Updateに接続できないなど、上記操作で設定できなかった場合に実施する方法です。
    この操作では対象OSのインストールディスクが必要です。

 

コントロールパネルから.NET Framework 2.0を利用できるようにする

  1. コントロールパネルの[プログラム]-[Windowsの機能の有効化または無効化]を選択します。
  2. [Windowsの機能]ダイアログにて[.NET Framework 3.5(.NET 2.0および3.0を含む)]にチェックします。
  3. [OK]をクリックし、ダイアログを閉じます。

 

コマンドプロンプトから.NET Framework 2.0を利用できるようにする

  1. 対象OSのインストールディスクを用意します。
  2. インストールディスクをディスクドライブに入れます。
  3. コマンドプロンプトで以下のコマンドを入力します。
    ※コマンドはディスクドライブがDドライブの場合を想定しています。環境に合わせて"d:"の箇所を変更してください。

DISM /Online /Enable-Feature /FeatureName:NetFx3 /All /LimitAccess /Source:d:\sources\sxs
