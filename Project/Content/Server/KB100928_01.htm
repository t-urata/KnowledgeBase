TimeTracker FX 3.9へのアップグレードを契機にSQL Serverのアップグレードを行う場合の手順をご紹介します。
ここでは、SQL Server 2014へのアップグレード例をご紹介します。

サーバマシンの移行とTimeTracker FX 3.9へのバージョンアップを同時に行う場合はこちらをご覧ください。

アップグレード手順

  1. TimeTracker FXのサービス停止連絡を行います。

  2. Server ManagerからTimeTracker FXで利用しているデータベースの登録を解除します。

    • データベースの登録解除方法はServeManagerヘルプのこちらをご覧ください。

  3. データベースファイル（mdfファイル）を任意の場所へ退避します。

    • デフォルトではSQL Serverのインストールフォルダに格納されています。

  4. SQL Serverをアンインストールします。

  5. SQL Server 2014をインストールします。インストール手順はこちらをご参照ください。

  6. TimeTracker FXをアップグレードします。アップグレード方法はこちらをご参照ください。

    • アップグレードは、前バージョンがインストールされているマシンに上書きでアップグレードが可能です。

  7. Server Managerを起動し、手順3で退避したデータベースを登録します。

    • データベースの登録方法はServeManagerヘルプのこちらをご覧下さい。

  8. 登録したデータベースのバージョンをアップグレードします。

    • アップグレード方法はServeManagerヘルプのこちらをご覧下さい。

  9. 手順7の際に作成したサーバ設定ファイルをクライアントユーザへ展開します。

  10. サービス開始の連絡を行い、TimeTracker FXの運用を開始します。

 

以上でアップグレード完了です。
