内容

TimeTracke FXは、SQL Server認証を利用するため、MSDE 2000が混合モード（Windows認証＋SQL Server認証）に設定されている必要があります。

既存のMSDE 2000をご利用される場合は、以下の手順でご確認ください。

 

確認方法

 

  1.  メニューから、[スタート] - [ファイル名を指定して実行] を選択してください。

  2. [ファイル名を指定して実行] ダイアログが表示されます。

  3. [名前] エディットボックスに、regedit と入力して、[OK] ボタンをクリックしてください。

  4. [レジストリエディタ] ウィンドウが表示されます。

  5. レジストリエディタでの操作を誤るとシステムに重大な障害を与える可能性がございます。

  1. 左側のツリーのノードを展開して、下記パスのノードを選択してください。
     [ マイ コンピュータ ]
    　　[ HKEY_LOCAL_MACHINE ]
    　　　　[ Software ]
    　　　　　　[ Microsoft ]
    　　　　　　　　[ MSSqlserver ]
    　　　　　　　　　　[ MSSqlServer ]

    • インスタンスが設定されている場合は こちらをご覧ください。

       [ マイ コンピュータ ]

      　　[ HKEY_LOCAL_MACHINE ]

      　　　　[ Software ]

      　　　　　　[ Microsoft ]

      　　　　　　　　[ Microsoft SQL Serve ]

      　　　　　　　　　　[ <Instance Name> ]

      　　　　　　　　　　　　[ MSSQLServer ]

  1. 認証モードの確認します。
    (2)のフォルダの下にある [ LoginMode ]をダブルクリックして、 表示された値が設定されている認証モードとなります。

 

[LoginMode]の設定値と認証モードについて

  • 1 ならば［Windows認証］

  • 2 または 0 ならば［混合モード認証］（Windows認証とSQL Server認証）

  • 1であれば、TimeTracker FXは動作しません。

  • 2 または 0 であれば、TimeTracker FXが動作します。
