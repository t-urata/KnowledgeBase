サーバマシンの移行とTimeTracker FX のバージョンアップを同時に行う

更新日:

2016/9/7

対象バージョン:

3.9

対象エディション:

Professional / Standard

 

概要

以下のような要因によって、TimeTracker FX運用中のサーバマシンを別のサーバマシンに移行することがあるかと思います。

  • TimeTracker FX のバージョンアップ
  • サーバマシンの老朽化やソフトウェア（OSなど）のサポート終了

場合によっては、移行先のサーバマシンの環境が利用中のTimeTracker FXの動作保証対象外となってしまうため、
サーバマシンの移行とTimeTracker FXのバージョンアップを合わせて検討する必要があります。

運用中のTimeTracker FXのバージョンや、移行元・移行先のサーバマシンの環境によって、サーバマシンの移行に必要な手順が異なります。
ここでは、運用環境に合わせたTimeTracker FX 3.9へのバージョンアップとサーバマシンの移行手順を紹介します。

  1. ご利用中のTimeTracker FXの動作保証環境については、こちらをご覧ください。
  2. TimeTracker FXのバージョンの確認方法については、こちらをご覧ください。

移行シナリオ

ここでは、移行シナリオの流れと手順の詳細を紹介します。
各移行シナリオの手順に従い、データベースを移行してください。

手順の詳細は、TimeTracker FXのヘルプやガイドに記載しております。
「ドキュメント」列のリンクよりドキュメントを開き、「実施範囲」列の内容を実施してください。

シナリオ1

一番基本的な移行手順となります。
移行元マシンにあるデータベースを移行先マシンにコピーし、最新バージョンで利用できるようアップグレードを行います。

シナリオ3

SQL Server の制約上、シナリオ1のように移行元マシンのTimeTracker FXのデータベースを移行先マシンの環境に登録できず、
シナリオ2のように移行元マシンにてTimeTracker FXをアップグレードできない場合の手順です。

中間マシンを用意し、データベースを中間マシン上でアップグレードすることで、移行先マシンにデータベースをコピーすることができます。
中間マシンとして以下の環境をご用意ください。

  • OS：　最新バージョンのTimeTracker FXおよびSQL Server の動作保証環境となるOS
  • SQL Server： 2008または2008 R2
