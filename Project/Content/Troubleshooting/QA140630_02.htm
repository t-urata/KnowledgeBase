Web Editionへのログインに失敗します

更新日:

2014/7/1

対象バージョン:

3.7

TimeTracker FX Web Editionへのログイン時にエラーが発生し、ログインできない場合があります。
この時の対処方法を紹介します。

症状

TimeTracker FX Web Editionのログイン画面で以下のようなエラー画面が表示され、ログインができません。
画面に表示されるエラーの内容は、原因によって異なります。

本現象は以下の場面で発生します。(必ず発生するわけではありません)

  • Web サーバにTimeTracker FX Web Editionをインストールして、クライアントPCからWeb Editionにアクセスする場合

原因と対策

本現象が発生する主な原因を列挙します。
これらの内容が該当しないかご確認ください。

  1. Web.configの編集内容に誤りがある
  2. app.jsonに指定したURLに誤りがある
  3. MIMEの設定漏れ
  4. Web Editionのインストール漏れ

これらの原因が該当しない場合は、ご利用中のサーバ・クライアントの環境(OSやIISの種類、ネットワーク環境など)を含む、お問い合わせに必要な情報をこちらまでご連絡ください。

1. Web.configの編集内容に誤りがある

原因 対策
Web.configは手動で編集するため、追記するべき個所が異なったり、タイプミスによる入力間違いが起こりやすく、間違って編集されたWeb.configでは、Web Editionを正常に利用できません。 Web.configの設定内容を確認してください。
「Web.config編集ツール」を使用すると、元のWeb.configファイルから確実に更新できます。

 

2. app.jsonに指定したURLに誤りがある

原因 対策
Web Editionのインストールフォルダ内に存在するapp.jsonファイルにはWeb Editionが使用するWeb APIのサイト情報が含まれています。
このURL情報が正しくない場合、TimeTracker FXのアカウント情報が取得できず、ログインエラーが発生します。		 app.jsonファイルで設定するURLが適切か確認してください。
URLは以下の設定となります。

http://(サーバーマシン名)/(Web APIのアプリケーション名)/

Web APIのアプリケーション名は、Web APIのインストール時に設定した名前となります。
TimeTracker FX 3.7の場合、初期設定の値は「TimeTrackerFX3.7.0」となります。
URLの末尾に必ず「/ (スラッシュ)」を付けてください。

 

3. MIMEの設定漏れ

原因 対策
Web APIのサイト情報を記載したapp.jsonファイルを利用するために必要なMIMEの設定ができておらず、Webサーバーで拡張子「.json」のファイルが利用できないためにログインエラーが発生します。 WebサーバーでMIMEの設定を行い、Web Editionで拡張子「.json」のファイルを利用できるようにしてください。
設定方法については、TimeTracker FX Web Edition セットアップガイドの「2.2 セットアップ手順詳細」をご確認ください。

 

4. Web Editionのインストール漏れ

原因 対策
インストーラー内にあるWeb Edition プログラムフォルダ内のファイルが一つでも欠けた場合、Web Editionを正常に利用できません。 インストールCD内にあるWeb Edition プログラムフォルダをフォルダごとコピーして、すべてのファイルを上書きコピーしてください。
