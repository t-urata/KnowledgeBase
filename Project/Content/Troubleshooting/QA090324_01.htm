タスクツリーが表示されません

タスクツリーは、表示しているタイムシートのアカウントに割り当てられたタスクを表示します。
タスクツリーにタスクが表示されない場合は、プランナーのタスクの設定などをご確認ください。

症状

プロジェクトやタスクが、［タスクツリー］に表示されない。

原因と対策

本現象が起こる可能性の高い原因から順にケース1～5として説明いたします、ケース1から順にご確認ください。

［ケース1］

【原因】

プランナーの情報は、タイムシートの起動時に読み込みます。
そのため、タイムシートの起動中にプランナーで更新したプロジェクト情報は、自動的に反映されません。

【対策と確認】

タイムシートの［ファイル］－［再読み込み］をクリックし、プランナーで編集された最新の情報を読み込みます。

ケース1で解決しない場合は、ケース2をご確認ください。

 

［ケース2］

【原因】

タスクに、リソースとして割り当てられていない。

【対策と確認】

以下の手順にて、リソースの割り当てを確認します。

  1. 該当タスクを含むプロジェクトを、［編集モード］で開きます。
  2. 該当タスクの、［リソース名］フィールドを確認します。
  3. ［リソース名］フィールドに割り当てられていない場合は、［リソース割り当て］を行います。
  4. ［保存］を実行します。
  5. タイムシートにて、［ファイル］－［再読み込み］をクリックします。

ケース2で解決しない場合は、ケース3をご確認ください。

 

［ケース3］

【原因】

［マイプロジェクト］で、表示されないタスクを含むプロジェクトのチェックが外れている。

【対策と確認】

  1. タイムシートにて、［ファイル］－［マイプロジェクト］をクリックします。
  2. 該当プロジェクトの［対象］フィールドに、チェックがされているか確認します。
    • ［すべてロードする］にチェックがされている場合は、このケースは該当しません。
  3. 該当プロジェクトの［対象］フィールドのチェックが外れている場合、チェックをいれます。
    • タスクを割り当てられたすべてのプロジェクトを表示する場合には、［すべてロードする］にチェックをいれます。
  4. ［OK］をクリックし、[マイプロジェクト]ウィンドウを閉じます。
  5. ［保存］を実行します。
  6. ［ファイル］－［再読み込み］をクリックします。

ケース3で解決しない場合は、ケース4をご確認ください。

 

［ケース4］

【原因】

プランナーで編集した内容が、保存されていない。

【対策と確認】

  1. 該当プロジェクトを、［編集モード］で開きます。
    • 既に開いている場合は、開き直してください。
  2. プロジェクトが編集した通りになっているか、確認します。
  3. 編集した内容がない場合は、再度編集します。
  4. ［保存］を実行します。
  5. タイムシートにて、［ファイル］－［再読み込み］をクリックします。

ケース4で解決しない場合は、ケース5をご確認ください。

 

［ケース5］

【原因】

［タスクツリー］に、タスク検索機能が適用されている。

    • ［キーワード］は、TimeTracker FX 2.5以降の機能です。

【対策と確認】

以下の内容に変更し、［タスクツリー］を確認します。

  • ［フィルタ］　：「割り当てられたタスク」を選択します。
  • ［グループ］　：「なし」を選択します。
  • ［キーワード］　：入力されている場合は、キーワードを削除します。
