このトピックの対象

TimeTracker FX 2.5.0以降

 

内容

SQL Server 2005で既定フォルダをドライブの直下（例えば、C:\やD:\等）に指定した場合、サンプルデータを含むデータベースを新規作成するとエラーが発生し、データベースの作成に失敗します。

 

  1. SQL Server Management StudioまたはSQL Server Management Studio Expressを使用して、既定フォルダをドライブの直下に指定する。

  2. Server Managerを使用して、サンプルデータを含むデータベースを新規作成する。

  3. エラーダイアログを表示し、データベースの作成に失敗する。

 

回避方法

SQL Server Management StudioまたはSQL Server Management Studio Expressを使用して、既定フォルダをドライブの直下以外に指定してください。
