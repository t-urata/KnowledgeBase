このトピックの対象

TimeTracker FX 2.5.0以降

 

内容

データファイルやログファイルに対して、mdfやldf以外の拡張子を設定した場合、Server Managerのデータベースのプロパティ画面で、データベースのファイル名が［取得失敗］となり、表示できません。

  1. Server Managerのデータベースのプロパティ画面で［データベースをコピーする］を選択する

  2. データベースコピーウィザードの新規データベース名入力画面で［データベースのファイル配置を個別に指定する］をチェックする

  3. 例として、配置場所の選択で、データファイルの拡張子をtxtにして設定する

  4. 例として、3と同様に、ログファイルの拡張子をwavにして設定する

  5. データベースコピーウィザードを進め、データベースをコピーする

  6. コピーしたデータベースのプロパティ画面で、データファイルとログファイルの場所が、［取得失敗］と表示する。

 

回避方法

以下の手順を行い、データベースを再登録してください。

  1. ［データベースの登録を解除する］を選択し、データベースを解除します。

  2. 解除したデータベースのファイルの拡張子を変更します。

  • データベースファイルの場合は、拡張子をmdfに変更します。

  • データベースログファイルの場合は、拡張子をldfに変更します。

  1. Server Managerのホーム画面で、［既存のデータファイルをサーバに登録する］を選択します。

  2. 2で拡張子を修正したファイルを選択します。

  3. データベースの登録ウィザードを先に進め、データベースを登録します。
