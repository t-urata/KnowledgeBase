内容

SQL Server 2005で、データベースを格納する既定のフォルダを、ルートフォルダ（例えば C:\ ）に設定すると、Server Managerでデータベースのコピーに失敗する。

  1. SQL Server Management StudioまたはSQL Server Management Studio Expressで既定フォルダをルートフォルダに指定する。

  2. Server Managerで任意のデータベースのプロパティ画面を表示する。

  3. ［データベースをコピーする］を選択する。

  4. データベースコピーウィザードで［データベースのファイル配置を個別に指定する］にチェックを入れずに、コピーする。

  5. データベースコピー中に、エラーが発生する。

 

回避方法

以下のいずれかの方法で、回避してください。

 

【回避方法1】

SQL Server Management StudioまたはSQL Server Management Studio Expressで既定フォルダをルートフォルダ以外に設定してください。

 

【回避方法2】

Server Managerのデータベースコピーウィザードで［データベースのファイル配置を個別に指定する］にチェックを入れて、ルートフォルダ以外にデータベースをコピーするように設定してください。
